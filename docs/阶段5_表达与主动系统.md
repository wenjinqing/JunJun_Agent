# 阶段 5 · 表达与主动系统

> 周期：2-3 天
> 目标：情绪/表情包/表达学习三件套 + 主动私聊/提醒/复读/统计——君君「活」起来。
> 修订（2026-07-16）：由原「阶段3」拆出；补 expression selector/reflector、steal_emoji、statistics 任务（原计划遗漏）；修正 repeat 语义（主动参与复读，非防复读）。

## 一、技术要点

### 1.1 情绪 `junjun_express/mood.py`
- 对齐原 `mood/mood_manager.py`：`ChatMood` 按会话维护情绪文本描述；新消息触发重评（`utils_small`），超时衰退（`regress_mood_prompt` 语义）。
- `enable_mood`、`emotion_style` 配置对齐；情绪状态接入 persona prompt（阶段 3 占位）。
- skill：`manage_mood`（agent 可读可调）。

### 1.2 表情包 `junjun_express/emoji.py`
- 对齐原 `emoji_system/emoji_manager.py`：
  - 库管理：`data/emoji/`（收集）→ `data/emoji_registed/`（VLM 描述+情感分类注册），Emoji 表（hash 去重、usage_count）。
  - **steal_emoji**：群里出现的表情包按配置偷图入库；`max_reg_num=2000` 上限，`do_replace` 满了换血。
  - 定期扫描注册循环（`check_interval` 10 分钟）。
  - 发送选择：情感 → 候选池 → 随机（对齐原 EmojiAction 语义），`emoji_chance=0.35` 概率参与。
- skill：`send_emoji`（描述→匹配→发送，独立冷却 60s/会话）。

### 1.3 表达学习三件套 `junjun_express/expression/`
- `learner.py`：对齐原 `ExpressionLearner`——LLM 抽取群友表达方式，embedding 去重，Expression 表按 chat 存储；`learning_list` 按会话配置强度。
- `selector.py`（**原计划漏了它，学了不用等于白学**）：回复生成前按语境选适配表达（对齐 `expression_selector.select_suitable_expressions`），注入 persona prompt；`expression_groups` 跨会话共享。
- `reflector.py`：向管理员（`reflect_operator_id`）求证表达是否恰当（低优先，可后补）。

### 1.4 主动系统 `junjun_agent/loop`
- 统一任务框架 `scheduler.py`：合并原 `AsyncTask`（interval 后台任务）与 `ScheduledTask`（cron 定时）两套为一个调度器，支持 interval/cron 两模式。
- **proactive_chat**（对齐原 `proactive_chat_manager.py`，注意原配置 `enable_in_groups=false` 仅私聊）：定期扫会话空闲（`min_idle_minutes`）→ LLM 生成话题（`proactive_topics.json` 去重存最近 10 条）→ LLM 二判 → 发送；`silent_hours=02:00-06:00` + `max_daily_proactive` 限额。
- **reminder**（对齐原 `reminder_task_manager.py`）：`set_reminder`/`list_reminders`/`cancel_reminder` skill 写 ReminderTasks 表；60s 轮询到期任务 → LLM 拟人化提醒文案 → 发送；`repeat_type` 周期任务；**启动时 `load_pending_reminders()` 恢复**。
- **repeat 主动复读**（对齐原 repeat_plugin，语义勘误：这是「参与复读」动作）：群内连续 `threshold=4` 条相同消息 → 冷却允许 → 君君跟一条；类级记录防重复跟（`min_interval_seconds=60`）。
- **statistics**（原计划遗漏）：OnlineTimeRecordTask、StatisticOutputTask 迁入调度器，数据供阶段 7 WebUI。

### 1.5 通用 skill 补全 `junjun_skills/builtin`
- `get_weather` / `web_search`（对齐原工具实现的 API 源）/ `send_message`（定向发送，含跨会话——供提醒/主动系统与 agent 共用同一发送出口）/ `send_poke`（戳一戳，原 `enable_poke` 配置开关）。

## 二、避坑与潜在 Bug

| 风险 | 说明 | 对策 |
|------|------|------|
| 主动发言打扰 | 夜间/高频主动被骂 | silent_hours + 日限额 + 仅私聊（对齐原配置）|
| 复读风暴 | 君君跟复读后又触发自己计数 | 自消息不计入复读检测；跟读后进冷却 |
| 表情包库爆炸 | steal 无限增长 | `max_reg_num` 上限 + `do_replace` 换血 |
| VLM 注册开销 | 每图都调 VLM | hash 去重，已注册跳过；注册循环限速 |
| 表达学习学脏话 | 学到不当表达 | learner 侧 LLM 过滤 + WebUI 可删（阶段 7）|
| selector 拖慢回复 | 每次回复都全表检索 | 表达按 chat 分组缓存 + embedding top-k 限 5 |
| 提醒重启丢失 | 进程重启未恢复 | `load_pending_reminders()` 启动恢复，单测覆盖 |
| 提醒时区错乱 | 时间解析歧义 | 统一本地时区 Asia/Shanghai，入库存 timestamp |
| 调度任务异常静默死亡 | 一个任务崩全部停 | 每任务独立 try/except + 崩溃自动重启 + WARN |
| 情绪评估烧 token | 每条消息都评 | 跟随 gate 结果：仅进入 L3 的消息触发重评 |
