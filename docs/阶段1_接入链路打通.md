# 阶段 1 · 接入链路打通

> 周期：1-2 天
> 目标：迁移 NapCat Adapter + 实现最小网关，QQ 消息能进能出。

---

## 一、技术要点

### 1.1 Adapter 层 `junjun_adapter_napcat`
- **复用** `maim_message` 库（`Router`/`RouteConfig`/`TargetConfig`），对齐原 `MaiBot-Napcat-Adapter/src/mmc_com_layer.py`。
- `com_layer.py`：建立 WS 连接，`router.register_class_handler(send_handler.handle_message)`，`await router.run()`。
- 配置（`config.toml` 迁移）：
  - `napcat_server`：host/port/token（指向 `E:\MaiM\NapCat.Shell` 的 WS，默认 8095）。
  - `maibot_server`：host/port（君君网关 WS 端口，原 8091，可改）。
- `recv_handler/`：迁移原 `message_handler.py`/`notice_handler.py`/`meta_event_handler.py`，把 OneBot 事件转成 `maim_message` 的 `UserInfo`/`GroupInfo`/`MessageToSend`。
- `send_handler/`：迁移原 `main_send_handler.py`/`nc_sending.py`，接收 Agent 产出的 `ReplySet`，调用 NapCat OneBot API（send_msg/send_poke 等）。

### 1.2 网关层 `junjun_core/gateway`
- `MessageRouter`：按 `platform:chat_id` 路由到 `ChatSession`（原 `message_router.py` 简化版，去掉多 bot 实例先做单 bot）。
- `ChatSessionManager`：维护会话→Agent 句柄映射（此阶段 Agent 用 echo 占位）。
- 黑白名单（迁移原 `config.toml [chat]`）：`group_list_type`/`private_list_type`/`ban_user_id`/`ban_qq_bot`。
- 速率限制：简单令牌桶，防刷屏。
- 入口：`async def handle_inbound(msg: UnifiedMessage) -> Optional[ReplySet]`。

### 1.3 数据契约冻结
- **必须先冻结** `UnifiedMessage`/`UserInfo`/`GroupInfo`/`MessageToSend`/`ReplySet` 的字段，再迁 Agent，否则收发错位。
- 字段对齐原 `maim_message` 协议，不擅自改名。

## 二、避坑与潜在 Bug

| 风险 | 说明 | 对策 |
|------|------|------|
| WS 断线重连 | NapCat 重启或网络抖动 | `maim_message` 内置重连，但要加日志和指数退避；监控 `on_disconnect` |
| 消息 ID 回环 | 原项目有 `message_id_echo` 自定义 handler | 迁移 echo 逻辑，避免 Agent 把自己发的消息当输入 |
| 群/@ 解析错误 | OneBot `array` 格式消息段解析 | `messagePostFormat=array`（见 NapCat 配置），用 `napcat` 消息段解析器；@ 段提取 `qq` 字段 |
| 黑白名单误杀 | whitelist 模式下漏配群导致全静默 | 启动时打印生效名单，被拦截消息打 DEBUG 日志 |
| 多 bot QQ 号混用 | 原 NapCat 配了多个 `napcat_*.json`（两个 QQ） | 单 bot 先跑通，多 bot 留到阶段 3 |
| 心跳超时 | NapCat 心跳 30s，Adapter `heartbeat_interval` 不一致导致判定离线 | 对齐 `heartbeat_interval=30`，超时阈值设 90s |
| poke 处理 | 原 `enable_poke` 已禁用但代码仍在 | 明确禁用则不注册 handler，避免无效分支 |
| 端口冲突 | 8091/8095 被原项目占用 | 新网关用新端口（如 8092），Adapter `maibot_server.port` 同步改 |

## 三、验收标准

- [ ] NapCat 启动后，Adapter 日志显示 WS 连接成功。
- [ ] QQ 发消息，网关收到 `UnifiedMessage` 并打印结构化日志。
- [ ] 网关 echo 回复，QQ 收到原样文本。
- [ ] 黑名单/白名单生效：非名单内群不回复。
- [ ] 断开 NapCat，Adapter 自动重连。
