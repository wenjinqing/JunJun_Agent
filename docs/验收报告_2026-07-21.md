## 三、发现的问题清单（按严重度分级）

### A. 工程基线（阻塞阶段7验收标准，必修）

- **A-1** `uv run` 不可用：根包与 `junjun_agent` 同名 `junjun-agent`。修根包 `name`（如 `junjun`）即可。
- **A-2** `ruff check` 32 错未清零：E402×24（sys.path hack 处加 `# noqa: E402` 或 ruff per-file-ignores）、F401×8（删未用 import）。
- **A-3** `models.py` 中文乱码（HEAD 已存在）：重写 docstring/日志文案。

### B. 计划内功能缺口（回归记录未披露，需补或确认放弃）

- **B-1 戳一戳全链路缺失**：入站 poke 解析（总目标#1）+ `send_poke` skill（阶段5）都没有。
- **B-2 通用 skill 缺 3 件**：`get_weather`、`send_message`（跨会话定向发送出口）、`send_poke`。阶段5交付物明确列出。
- **B-3 配置热改无事件通知**：WebUI `set_config` 只写内存 dict（`webui/server.py:112-127`），Agent/persona 不重载——**真机验收清单「WebUI 改 talk_value → 行为立即变化」将失败**，阶段7验收标准之一。
- **B-4 WebUI 缺 2 个计划路由**：plugin 管理（启用/禁用）、chat 调试（查看上下文/记忆）。
- **B-5 trace_id 未写结构化日志**：阶段2（trace_id 写入日志）与阶段7（日志页跳转 Langfuse）均落空。
- **B-6 Timing Gate 未实装**（阶段3，`enable_timing_gate` 默认关——计划虽默认关，但属于交付物）。
- **B-7 Gateway 速率限制未实装**（阶段1 交付物「简单令牌桶防刷屏」）。
- **B-8 OnlineTimeRecordTask 未迁移**（阶段5 statistics 应含两件，只迁了 StatisticOutputTask）。
- **B-9 `query_chat_history` skill 缺**（阶段4 记忆类 6 件之一，实际 5 件+learn_jargon）。
- **B-10 VLM 入站识图未实装**（回归记录已如实标 ⬜，依赖 vlm 任务槽，见 C-1）。
- **B-11 合并转发解析未实装**（回归记录已如实披露，原配置 parseMultMsg=false，低优先）。
- **B-12 短期记忆未升级 LangGraph checkpointer**（阶段4 计划），仍为内存窗口+每会话 agent 实例兜底——影响小，但 `short_term.py:4` 注释自己写着「阶段4升级」却没做。

### C. 与计划/记录不符（文档或配置层面）

- **C-1 模型任务槽大幅简化**：`model_config.toml` 只有 gate/agent/utils 3 槽，且**全部共用同一组 `LLM_BASE_URL/LLM_MODEL/LLM_API_KEY` env** → 阶段2验收「gate 与 agent 用不同模型」「切换 deepseek/siliconflow 两 provider」按当前配置无法满足；计划要求的 model_list 顺序 fallback、vlm/utils_small/embedding/lpmm 槽均未补。这是与计划偏离最大的一处。
- **C-2** 回归记录称「内置 12 skill」，实际注册 **14 个**（记录数字不准，功能不少）。
- **C-3** README 架构图为 ASCII，阶段7交付物要求 Mermaid。
- **C-4** `bot_config.toml.example` 的 `silent_hours = "23:00-09:00"`，计划对齐原配置 `02:00-06:00`。
- **C-5** LLMUsage 索引字段名 `time` 而非文档说的 `created_at`（功能等效，文档改一下即可）。

### D. 已如实披露、可接受的偏离

- WebUI 前端重写（理由充分：旧前端服务已砍功能，负资产）——保留偏离。
- LPMM / reflector / hippo 三项曾偏离已补齐，180 测试覆盖。
- ASR 预留不实装（原配置即关闭）。

### E. 未提交改动状况（进行中工作，建议先收尾再验收）

- 6 个修改文件 + 未跟踪 `langfuse/`（docker-compose，计划中产物 ✅）和 `langfuse-skills-tmp/`（**临时目录，不应入库，且贡献了 1 个 ruff 错误**）。
- `models.py` 新增 `http_socket_options`：已验证是 langchain-openai 合法字段（keepalive 调优，合理）；但同处新增的 `asyncio.get_event_loop()` hack 值得审视（`get_chat_model` 是同步函数，为何需要事件循环？疑似掩盖别的问题）。
- 这批改动新引入 6 个 F401，需一并清理。
