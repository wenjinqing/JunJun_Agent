# JunJun_Agent 使用指南

> QQ 聊天机器人「君君」——由原 MaiBot 升级为 **LangChain 1.x Agent 架构**。
> 本文档：架构简介 + 启动方式 + 全部命令/插件/工具/MCP 使用说明。

---

## 一、这是什么

一个有人设的 QQ 群聊/私聊 bot。核心特点：

- **三级决策漏斗（省 token）**：每条消息先过 0-token 规则门，再过小模型语义门，最后才进主 Agent——大部分闲聊在免费阶段就被滤掉
- **LangChain 1.x Agent**：主决策是带工具的 Agent（LangGraph runtime），自己决定什么时候说话、什么时候调工具、什么时候闭嘴
- **人格化**：人设/回复风格/情绪系统/错别字/分条发送/复读跟读/表情包，全套拟人化
- **记忆体系**：短期窗口 + 话题摘要 + 长期向量记忆 + LPMM 知识图谱 + 用户画像 + 好感度
- **安全体系**：硬编码管理员机制 + 通道 token 鉴权 + 工具层权限门 + prompt 防注入 + 越权自动上报

```
NapCat(QQ) ⇄ Adapter(8095) ⇄ 网关(8092, token 鉴权) ⇄ 决策漏斗
                                                    ├─ 命令总线（0 token：/cmd、关键词）
                                                    ├─ 拦截器（0 token：链接自动解析）
                                                    ├─ L1 规则门（概率/@/沉默模式）
                                                    ├─ L2 语义门（小模型）
                                                    └─ L3 主 Agent（工具：内置18 + 插件 + MCP）
```

## 二、启动方式

```bash
# 1.（可选）Langfuse 追踪面板
docker compose -f langfuse/docker-compose.yml up -d   # http://localhost:3000

# 2. 主程序（网关 + 决策 + WebUI）
uv run scripts/run_junjun.py

# 3. Adapter（另开终端）
uv run scripts/run_adapter.py

# 4. NapCat（QQ 本体）
E:\MaiM\NapCat.Shell\启动君君.bat
```

配置文件速查：

| 文件 | 内容 |
|---|---|
| `.env` | 所有密钥/地址（gitignored）：模型 API、管理员 QQ、通道 token、插件 key |
| `config/bot_config.toml` | 主配置：人设/发言频率/黑白名单/各功能开关（逐项中文注释，WebUI 可热改） |
| `config/model_config.toml` | 模型任务槽：gate/agent/utils/utils_small/vlm 各用什么模型 + 降级链 |
| `config/mcp_servers.toml` | MCP 外部工具 server 声明 |
| `junjun_adapter_napcat/config.toml` | Adapter：NapCat/网关地址与 token |
| WebUI `http://127.0.0.1:8002` | 配置热改/日志/统计/插件开关（需 `.env` 设 `WEBUI_ENABLED=true` + `WEBUI_TOKEN`） |

## 六、MCP 外部插件

支持标准 **MCP（Model Context Protocol）** server，工具自动转为 LangChain 工具注入 Agent，命名空间 `mcp_<server>_<tool>`。

**当前已接入**（`config/mcp_servers.toml`，全部冒烟验证通过）：

| Server | 工具数 | 能力 | 启动 |
|---|---|---|---|
| `relationship`（自研） | 5 | 印象/标签/称呼/惩罚 | python -m（内置） |
| `trends` | 21 | 微博/知乎/B站/抖音/36氪 等 20+ 中文热榜 | `npx mcp-trends-hub` |
| `train12306` | 8 | 火车票余票/中转/经停查询 | `npx 12306-mcp` |
| `bilibili` | 8 | B站搜索/热门榜/视频详情/UP主 | `npx bilibili-mcp-js` |
| `fetch`（官方） | 1 | 任意网页 → markdown | `uvx mcp-server-fetch` |
| `wikipedia` | 22 | 中文维基百科搜索/摘要/全文 | `uvx wikipedia-mcp` |
| `douyin_extract` | 5 | 抖音链接→无水印下载+语音转文案 | `uvx douyin-mcp-server` |
| `amap`（待启用） | 12 | 高德天气/POI/路线 | `npx @amap/amap-maps-mcp-server` |

> `time` server 与内置 get_time 重复，已禁用（2026-07-22 权限治理）。

- 环境要求：npx 系需 **Node ≥ 22**（本机已用 nvm 升到 22.23.1）；uvx 系需 uv
- `amap` 默认 `enable = false`：去 [高德开放平台](https://lbs.amap.com) 申请免费 Web 服务 key，填 `.env AMAP_MAPS_API_KEY` 后把 toml 里 `enable` 改 `true`
- env 值支持 `"${VAR}"` 占位符（从 .env 替换，密钥不落 toml）

**添加新 MCP server**（以 stdio 为例）：

```toml
# config/mcp_servers.toml
[servers.my_server]
enable = true
command = "cmd"                                    # Windows 下 npx 必须 cmd /c 包一层
args = ["/c", "npx", "-y", "some-mcp-server"]
env = { SOME_KEY = "${SOME_KEY}" }                 # key 放 .env，toml 只写占位符
# uvx/Python 系直拉：command = "uvx", args = ["some-server"]
```

重启后自动连接（10s 超时，单个失败降级跳过不阻塞启动）。

## 八、安全体系与权限矩阵（管理员 = 你）

- `.env ADMIN_QQ`：唯一最高权限管理员，**硬编码机制**——配置/WebUI/聊天内容都改不了
- **权限语义（2026-07-22 用户指定）**：管理员平时就是普通群友——走正常决策漏斗、
  不直通、不特殊对待，君君当你是好朋友而不是上级。**只有 @君君 给出明确指令**
  （或私聊直接说）时，管理员权限才激活（敏感操作/🔒 命令放行）
- 越权行为 → 自动拒绝 + **私聊上报管理员**（谁、在哪、想干什么）
- 通道鉴权：NapCat⇄Adapter⇄网关全部 token 认证，伪造消息进不来
- 防注入：聊天里自称管理员/贴"系统指令"一律无效（只认真实 QQ + @激活）；bot 不会泄露配置和密钥

## 九、常见问题

| 问题 | 解决 |
|---|---|
| `/draw` 说没配 key | `.env` 加 `MODELSCOPE_API_KEY` |
| B 站只发信息卡没视频 | 装 ffmpeg 到 PATH（`winget install Gyan.FFmpeg`），重启 bot |
| `抽老婆` 说拿不到群成员 | NapCat 开 httpServer 并配 `.env NAPCAT_HTTP_BASE` + `NAPCAT_HTTP_TOKEN` |
| 群里 bot 完全不理人 | 检查 `[chat] group_list_type`：whitelist 模式下 `group_list` 为空 = 全不回 |
| 图片只显示 [图片] | 配 `.env VLM_*` 三件套（识图模型） |
| 提醒时间不对 | 已修复（2026-07-21 回归），确认 bot 是本仓库最新 |
| WebUI 打不开 | `.env` 设 `WEBUI_ENABLED=true` 和 `WEBUI_TOKEN` |
